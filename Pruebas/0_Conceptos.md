# 1. ¿Qué son las pruebas (testing) en software?

Las pruebas son procesos para verificar que un sistema:

- Funciona correctamente
- Cumple los requisitos
- No tiene errores críticos
- Se comporta como se espera
- Es seguro y confiable

# 2. ¿Por qué son importantes las pruebas?

Porque reducen:

- Bugs en producción
- Fallos de seguridad
- Costos de mantenimiento
- Riesgos en despliegues
- Problemas con usuarios

Y aumentan:

- Calidad del software
- Confianza en el sistema
- Velocidad de desarrollo
- Facilidad de cambios

# 3. Tipos de pruebas (visión general)

| Tipo / Técnica   | Qué verifica                           | Quién la hace |
| ---------------- | -------------------------------------- | ------------- |
| Unitarias        | Una función o método                   | Desarrollador |
| Integración      | Componentes juntos                     | Desarrollador |
| Sistema          | Todo el sistema                        | QA / Dev      |
| Aceptación       | Requisitos del negocio                 | Cliente / QA  |
| API              | Endpoints HTTP                         | Dev / QA      |
| End-to-End (E2E) | Flujo completo                         | QA / Dev      |
| Performance      | Velocidad y carga                      | DevOps / QA   |
| Seguridad        | Vulnerabilidades                       | Seguridad     |
| Regresión        | Que nada se haya roto                  | QA / Dev      |
| Mocking          | Simulación de dependencias             | Dev           |
| CI/CD            | Automatización de pruebas y despliegue | DevOps        |
