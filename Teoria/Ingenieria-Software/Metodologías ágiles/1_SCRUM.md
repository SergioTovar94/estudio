# ¿Qué es Scrum?

Scrum es un marco de trabajo ágil que organiza el desarrollo en ciclos cortos llamados sprints.

## 1 Se crea un backlog

Backlog es una lista priorizada de cosas que se deben construir o mejorar.

- Crear login
- Crear registro
- Conectar API
- Validar formulario

## 2 Sprint

Normalmente dura dos semanas.
Daily Scrum: cada día se hace una reunión de 15 minutos, cada persona dice qué hizo y se rectifican cosas.

## 3 Al final del Sprint

- Se muestra lo que se hizo
- Se revisa si funciona
- Se mejora el proceso

## Roles en SCRUM

| Rol              | Qué hace                         |
| ---------------- | -------------------------------- |
| Product Owner    | Decide qué se construye          |
| Scrum Master     | Ayuda a que el equipo siga Scrum |
| Development Team | Los devs que programan           |

# Ceremonias de Scrum

Scrum tiene 5 eventos (ceremonias) principales. Para tu rol, necesitas saber qué pasa en cada una y cómo participas.

| Ceremonia                                                           | Propósito                                                                  | Tu rol como analista                                                                                                                    |
| ------------------------------------------------------------------- | -------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| Sprint Planning                                                     | Definir qué se hará en el sprint y cómo.                                   | Explicar las historias de usuario con mayor prioridad, resolver dudas de negocio, asegurar que los criterios de aceptación sean claros. |
| Daily Scrum                                                         | Sincronización de 15 min. Qué se hizo ayer, qué se hará hoy, impedimentos. | Escuchar impedimentos técnicos que puedan afectar los requerimientos; ofrecer ayuda para despejar dudas con el negocio.                 |
| Sprint Review                                                       | Demostrar lo desarrollado a los interesados.                               | Validar junto al Product Owner que lo entregado cumple los criterios de aceptación; recoger feedback.                                   |
| Sprint Retrospective                                                | Mejora continua del equipo.                                                | Identificar qué falló en la comunicación de requerimientos y proponer mejoras.                                                          |
| Refinamiento del backlog (no es ceremonia oficial pero se practica) | Preparar historias para futuros sprints.                                   | Analizar requerimientos entrantes, desglosarlos, detallarlos y estimarlos con el equipo.                                                |
