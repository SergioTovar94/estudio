# PMI

PMI (Project Management Institute - Instituto de Gestión de Proyectos) es la organización que publica la guía PMBOK (Project Management Body of Knowledge - Cuerpo de conocimientos de gestión de proyectos). No es una metodología en sí misma, sino un estándar que recopila buenas prácticas para la dirección de proyectos.

# Contenido clave del PMBOK (para tu rol)

| Grupo de procesos   | ¿Qué implica para ti?                                                                                                                        |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| Inicio              | Desarrollar el acta de constitución (Project Charter) e identificar a los interesados.                                                       |
| Planificación       | Definir alcance, crear EDT (WBS), desarrollar cronograma, estimar costos, planificar riesgos. Aquí documentas los requerimientos en detalle. |
| Ejecución           | Dirigir y gestionar el trabajo del equipo. Tú acompañas al equipo técnico y facilitas la comunicación.                                       |
| Monitoreo y control | Controlar cambios, validar alcance, hacer seguimiento a cronograma y costos. Tu día a día de reportes y seguimiento.                         |
| Cierre              | Cerrar el proyecto o fase, obtener la aceptación formal, documentar lecciones aprendidas.                                                    |

## 1. Inicio - ¿Vale la pena hacer este proyecto?

- Objetivo
- Alcance
- Quién autoriza
- Quienes participan

Entregable:
Acta de constitución. Autoriza formalmente el proyecto. Nombra al sponsor, objetivo de alto nivel y recursos. | Documento de una página que firmas con el patrocinador.

## 2. Planificación - ¿Cómo lo vamos a hacer?

- Qué se va a hacer
- Cuánto tiempo tomará
- Cuánto costará
- Qué riesgos existen
- Quién hace qué

Entregables:

- WBS / EDT: (Work Breakdown Structure / Estructura de Desglose del Trabajo) - Una lista jerárquica que divide el proyecto en partes pequeñas y manejables.
- Cronograma: El calendario del proyecto. Define qué tarea se hace, cuando empieza, cuando termina y de qué depende.
- Plan de riesgos: Una lista de problemas que podrían pasar y qué harás si pasan.
- Plan de comunicación: Quién recibe información, qué información, cuándo y cómo.
- Requerimientos: Lo que el sistema debe hacer.

# Resumen: Artefactos PMI

| Artefacto                                     | Propósito                                                                               | Ejemplo concreto                                                                             |
| --------------------------------------------- | --------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| Project Charter (Acta de constitución)        | Autoriza formalmente el proyecto. Nombra al sponsor, objetivo de alto nivel y recursos. | Documento de una página que firmas con el patrocinador.                                      |
| EDT / WBS (Estructura de desglose de trabajo) | Divide el alcance en paquetes de trabajo manejables.                                    | “Módulo de autenticación → Desarrollo backend → API de login”                                |
| Cronograma (Diagrama de Gantt)                | Muestra fechas de inicio/fin y dependencias.                                            | Herramienta como MS Project o Excel con hitos y duraciones.                                  |
| Matriz de trazabilidad de requisitos          | Conecta cada requerimiento con su diseño, desarrollo, prueba y entrega.                 | Te lo explico en detalle más abajo.                                                          |
| Registro de riesgos                           | Lista de riesgos con probabilidad, impacto y plan de respuesta.                         | “Riesgo: retraso en aprobación del cliente → Mitigación: enviar actas con plazos definidos.” |
| Acta de cierre / Aceptación formal            | Documento donde el cliente acepta que el entregable cumple los criterios.               | A veces incluye lecciones aprendidas.                                                        |
