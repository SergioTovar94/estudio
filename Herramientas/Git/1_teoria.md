# Conceptos Git

Es un software de control de versiones en desarrollo y producción.

### Repositorio

Es el proyecto versionado por Git.

Puede ser:

- Local (máquina)
- Remoto (GitHub, GitLab)

### Snapshot

Es una fotografía del proyecto en un momento determinado.

### Commit

Un punto en la historia del proyecto. Cada commit tiene id, autor, fecha y mensaje.

### Rama

Línea paralela de trabajo.

### Remote

Repositorio conectado a un proyecto local

Git trabaja en tres niveles.

1. Directorio de trabajo:
2. Stagin Area: archivos que se preparan para el próximo commit.
3. Repository: cuando se hace commit git toma lo que está en staging y lo guarda como un snapshot. Esto signfica que es una instantánea del proyecto y sobre la cuál se puede devolver.
